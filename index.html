Pengenalan Cloud Computing

Memahami konsep dasar, fungsi, cara kerja, dan manfaat komputasi awan bagi kehidupan sehari-hari.

Apa itu Cloud Computing?

📖

Definisi Cloud Computing

Cloud Computing adalah gabungan pemanfaatan teknologi komputer dan pengembangan berbasis Internet. Disebut "Cloud" karena informasi diakses secara remote di "awan" (ruang virtual). Bisa diakses kapan saja asalkan ada koneksi internet, dan bersifat public atau private.

🌐

Public Cloud

Menyediakan layanannya secara publik di internet. Dapat diakses oleh siapa saja. Contoh: Google Drive, YouTube, Gmail.

UmumMurah
🔒

Private Cloud

Hanya menyediakan layanan kepada user tertentu saja. Lebih aman dan terkontrol. Contoh: cloud internal bank.

AmanEksklusif
🔀

Hybrid Cloud

Kombinasi public & private. Beberapa data di server privat, sisanya di public cloud. Fleksibel untuk perusahaan besar.

FleksibelOptimal

5 Fungsi Cloud Computing

✉️

Email

Layanan email berbasis cloud seperti Gmail dan Zimbra Mail. Bisa diakses dari perangkat mana saja.

💾

Penyimpanan & Backup

Menyimpan, backup, dan mengambil data secara remote. Tidak perlu hardisk fisik tambahan.

⚙️

Membuat & Mengetes Aplikasi

Developer bisa membuat, mengetes, dan men-deploy aplikasi tanpa infrastruktur fisik sendiri.

📊

Menganalisis Data

Menganalisis data dalam skala besar menggunakan computing power dari server cloud yang sangat besar.

▶️

Streaming Audio & Video

Streaming langsung dari cloud tanpa perlu menyimpan di perangkat lokal. Contoh: Spotify, YouTube, Netflix.

Cara Kerja Cloud Computing

⚙️

Dua Faktor Utama

Setiap varian Cloud Computing memiliki dua faktor yang sama: (1) data center yang berada di luar (remote) dan (2) harus memiliki internet untuk mengaksesnya. Sumber daya server dikumpulkan untuk membuat platform besar yang menampung layanan virtual. Resource diatur agar fleksibel sehingga pengguna bisa mengakses lebih banyak ruang penyimpanan kapanpun dibutuhkan.

Konsep Cloud Computing

Pelajari 7 konsep utama sistem komputasi awan yang harus dipahami secara menyeluruh.

7 Konsep Utama Sistem Komputasi Awan

💡

Mengapa Harus Dipelajari?

Banyak perusahaan yang tahu tentang cloud computing tetapi kurang memahami konsepnya. Selain fungsi, konsep-konsepnya juga harus dipelajari secara menyeluruh agar benar-benar paham cara menggunakannya.

1

☁️ Internet sebagai Pusat Kontrol

Perusahaan atau bisnis harus terkoneksi pada internet sebagai pusat pengendalian saat mengakses berbagai data dan dokumen. Tanpa internet, sistem cloud computing tidak bisa berjalan sama sekali.

2

📱 Perangkat Mempunyai Jaringan Internet

Setiap perangkat yang mengakses cloud harus mampu menerima sinyal internet. Perangkat yang rusak atau tidak bisa menerima sinyal akan menghambat proses pengaksesan data.

3

🌐 Bekerja Dalam Ruang Virtual Cloud

Data dan dokumen fisik harus diubah menjadi digital terlebih dahulu sebelum masuk ke virtual cloud. Ini merupakan syarat utama penggunaan cloud computing.

4

💾 Menyimpan Data dan Dokumen Digital

Cloud computing menyimpan data-data digital secara aman di dalam ruang virtual. Kemudahan pengolahan data menjadi dokumen digital adalah salah satu keuntungan utama cloud.

5

🔐 Mengamankan Data dan Dokumen

Penyedia jasa membuat software khusus yang hanya bisa diakses oleh masing-masing kliennya. Data dan dokumen terjaga sangat aman di dalam sistem virtual yang dibangun.

6

📤 Mentransfer Data dan Dokumen

Perusahaan yang menggunakan cloud dimudahkan dalam mentransfer data antar karyawan. Komputer-komputer terhubung satu sama lain melalui jaringan, meningkatkan kinerja perusahaan.

7

🚀 Tidak Membebankan Penggunaan Memori

Cloud computing tidak membebankan penggunaan memori lokal pada software perusahaan. Tidak perlu membeli hardisk tambahan karena storage tersedia di cloud secara elastis.

Layanan Cloud Computing

Memahami tiga model layanan utama: SaaS, IaaS, dan PaaS beserta karakteristik dan contohnya.

Tiga Model Layanan Utama

💻

SaaS — Software as a Service

Perangkat lunak yang dijadikan sebagai layanan online. Pengguna memakai aplikasi tanpa perlu memahami teknis penyimpanan, server, atau pengelolaan perangkat.

🎯 Contoh: Office 365, Google Workspace, Zoom

End UserSiap Pakai
🖥️

IaaS — Infrastructure as a Service

Layanan server fisik dan virtual server. Menyediakan resource cloud seperti server, jaringan, storage dan ruang data center tanpa membeli perangkat fisik.

🎯 Contoh: Microsoft Azure, Indonesian Cloud, AWS

Admin ITKendali Penuh
🔧

PaaS — Platform as a Service

Layanan cloud dalam bentuk platform untuk membuat aplikasi. Memberikan framework bagi developer. Server, storage, dan jaringan dikelola penyedia pihak ketiga.

🎯 Contoh: Salesforce, Heroku, Google App Engine

DeveloperFramework

Karakteristik Detail

💻 Karakteristik SaaS
  • ✅ Dikelola dari lokasi pusat — Software dikelola terpusat, pengguna tidak perlu memikirkan teknis.
  • ✅ Akses via internet — Memerlukan jaringan internet; kualitas pakai bergantung pada kualitas internet.
  • ✅ Pembaruan otomatis — Pembaruan hardware/software menjadi tanggung jawab penyedia, bukan pengguna.
🖥️ Karakteristik IaaS
  • ✅ Sumber daya sebagai layanan — Server, storage, dan jaringan tersedia sebagai layanan cloud.
  • ✅ Biaya bervariasi — Tergantung penggunaan perusahaan (pay-as-you-go).
  • ✅ Sangat scalable — Dapat dengan cepat disesuaikan dengan kebutuhan.
  • ✅ Multi-user — Dapat diakses oleh banyak pengguna dalam satu perangkat.
  • ✅ Kendali penuh — Perusahaan mempunyai kendali penuh atas infrastrukturnya.
  • ✅ Fleksibilitas tinggi — Bisa dikonfigurasi sesuai kebutuhan spesifik.
🔧 Karakteristik PaaS
  • ✅ Dikelola dari lokasi pusat — Platform dikelola secara terpusat oleh penyedia.
  • ✅ Akses via internet — Memerlukan koneksi internet yang stabil dan andal.
  • ✅ Server dikelola pihak ketiga — Semua server, penyimpanan, jaringan dikelola oleh penyedia.
  • ✅ Framework untuk developer — Menyediakan tools dan framework untuk membangun aplikasi yang disesuaikan.

BAB 4 · MODUL PRAKTIKUM OOP PHP

Object Oriented Programming

Belajar OOP PHP dari nol sampai bisa membuat sistem informasi mahasiswa lengkap dengan database.

Konsep Dasar OOP

📐

CLASS

Blueprint/cetak biru untuk membangun Object. Memodelkan objek dunia nyata ke dalam kode. Contoh: class Mobil memiliki karakteristik warna, bobot.

📦

OBJECT

Contoh spesifik (instance) dari sebuah Class. Dari class Mobil bisa dibuat object $myCar dan $yourCar.

🏷️

PROPERTIES

Karakteristik class yang menyerupai variabel. Memiliki nama dan nilai. Contoh: $warna, $berat.

⚙️

METHODS

Tindakan/kebiasaan yang berkaitan dengan class. Seperti fungsi, bisa menerima argumen dan mengakses properties object. Disebut juga 'anggota' class.

Tiga Jenis Modifier Akses

ModifierClass SendiriClass TurunanClass LainGunakan Saat...
public✅ Ya✅ Ya✅ YaProperty/method boleh diakses siapa saja
protected✅ Ya✅ Ya❌ TidakHanya class & turunannya yang boleh akses
private✅ Ya❌ Tidak❌ TidakHanya class itu sendiri yang boleh akses

📋 Tutorial OOP Langkah demi Langkah

1

Buat File Class (classLibrary.php)

Pertama, buat file PHP yang berisi definisi class. Beri nama class dengan huruf kapital. Tambahkan properties menggunakan var $nama.

classLibrary.php
<?php
class Orang {
  var $nama;
}
?>
💡 Class adalah "cetakan" atau blueprint untuk membuat object
2

Tambahkan Getter & Setter

Fungsi setter (setNama) untuk mengatur nilai. Fungsi getter (getNama) untuk mengambil nilai. $this menunjuk ke object saat ini.

classLibrary.php
<?php
class Orang {
  var $nama;

  public function setNama($nama) {
    $this->nama = $nama;  // atur nilai
  }

  public function getNama() {
    return $this->nama;  // ambil nilai
  }
}
?>
✅ $this adalah referensi ke object yang sedang aktif
3

Gunakan Class di File Lain (index.php)

Gunakan include atau require untuk memuat class. Buat object baru dengan new NamaClass(). Ini disebut instansiasi.

index.php
<?php
include("classLibrary.php");

// Buat object dari class Orang
$org = new Orang();

// Atur nilai dengan setter
$org->setNama("Budi Santoso");

// Ambil dan tampilkan dengan getter
echo "Nama: " . $org->getNama();
// Output: Nama: Budi Santoso
?>
4

Buat Class Koneksi Database

Class CConnection untuk menghubungkan aplikasi ke database MySQL. Memiliki fungsi openConnection() dan closeConnection().

CConnection.php
<?php
class CConnection {
  var $koneksi;

  public function openConnection() {
    $server   = 'localhost';
    $database = 'dbsatu';
    $user     = 'root';
    $password = 'password';
    $this->koneksi =
      mysql_connect($server, $user, $password);
    mysql_select_db($database);
  }

  public function closeConnection() {
    mysql_close($this->koneksi);
  }
}
?>
5

Buat Class Model (CMahasiswa.php)

Class model berisi properties data mahasiswa dan fungsi-fungsi CRUD (Create, Read, Update, Delete). Menggunakan class CConnection untuk akses database.

CMahasiswa.php
<?php
require("CConnection.php");

class CMahasiswa {
  private $nim, $nama, $jnskel, $agama;

  // Getter & Setter
  public function getNim() { return $this->nim; }
  public function setNim($nim) { $this->nim = $nim; }
  public function getNama() { return $this->nama; }
  public function setNama($nama) { $this->nama = $nama; }

  // Fungsi simpan ke database
  public function getInsert() {
    $sql = "INSERT INTO mahasiswa VALUES
      ('".$this->getNim()."','"
      .$this->getNama()."')";
    $c = new CConnection();
    $c->openConnection();
    $q = mysql_query($sql);
    $c->closeConnection();
    return $q;
  }

  // Fungsi tampilkan semua data
  public function getList() {
    $sql = "SELECT * FROM mahasiswa";
    $c = new CConnection();
    $c->openConnection();
    return mysql_query($sql);
  }
}
?>
